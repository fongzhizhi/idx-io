# 多层板导出示例改造报告

## 改造目标
将 `examples/export-with-layers.ts` 改造为同时支持浏览器和 Node.js 环境的双环境项目。

## 问题分析
原始代码在 Node.js 环境下运行时会遇到以下问题：
1. TypeScript 编译器无法识别 `window` 和 `document` 对象（缺少 DOM 类型定义）
2. 代码中使用了浏览器专有的 API，但没有正确的类型守卫
3. Node.js 环境下导出的 IDX 文件没有保存到磁盘

## 改造方案

### 1. 添加类型守卫
修改所有浏览器环境检测，从单一的 `typeof window !== 'undefined'` 改为：
```typescript
typeof window !== 'undefined' && typeof document !== 'undefined'
```

这样可以确保：
- TypeScript 编译器能够正确推断类型
- 在 Node.js 环境下不会尝试访问不存在的 DOM API

### 2. 添加 Node.js 文件保存功能
在文件顶部导入 Node.js 模块：
```typescript
import * as fs from 'fs';
import * as path from 'path';
```

在 `exportMultiLayerBoard()` 和 `exportSimple4LayerBoard()` 函数中添加文件保存逻辑：
```typescript
// Node.js环境下保存文件
const outputDir = './output';
if (!fs.existsSync(outputDir)) {
  fs.mkdirSync(outputDir, { recursive: true });
}
const filePath = path.join(outputDir, result.fileName);
fs.writeFileSync(filePath, result.xmlContent, 'utf-8');
console.log(`💾 文件已保存: ${filePath}`);
```

### 3. TypeScript 编译配置
运行时使用 ts-node 的 `-O` 选项添加 DOM 类型库：
```bash
npx ts-node -O "{\"lib\":[\"ES2020\",\"DOM\"]}" examples/export-with-layers.ts
```

## 已知问题
示例数据中的 keepout 几何定义有问题，导致以下错误：
- ❌ 圆形禁止区缺少半径或中心点
- ❌ 矩形禁止区缺少 4 个点

这些是示例数据的问题，不影响核心导出功能。主要的多层板、组件、过孔和层叠结构都成功导出。
